## 배경

서버리스 환경의 가장 큰 숙제 중 하나는 **Cold Start**다.
요청이 없다가 갑자기 몰리면 컨테이너를 새로 띄우는 시간이 SLA를 위협한다.

소프트뱅크 해커톤에서 이 문제를 정면으로 마주쳤을 때, 단순한 "미리 많이 띄워두기" 대신 **예측 기반 사전 배분**을 시도했다.

## 핵심 아이디어

```
트래픽 예측 → 필요 레플리카 수 계산 → 사전 스케일아웃
```

단순하지만, 예측이 틀리면 오히려 리소스 낭비가 심해진다.
그래서 두 가지를 결합했다:

1. **EMA (지수이동평균)**: 최근 QPS 트렌드를 부드럽게 추종
2. **시간대별 가중치**: 피크 타임(0.6~1.5)을 미리 알고 보정

## DeepSeek EPLB에서 배운 것

DeepSeek의 EPLB(Expert Parallelism Load Balancer)는 MoE 모델에서 전문가 간 부하를 균등하게 분산하는 알고리즘이다.
이 개념을 Kubernetes 레플리카 분산에 적용했다.

핵심은 **"예측의 정확도"보다 "예측 실패 시의 복구 비용"**을 더 중요하게 보는 것이다.

## 배운 점

> 분산 시스템에서 ML 예측을 쓸 때, 모델 정확도보다 **예측 주기와 실행 레이턴시의 트레이드오프**가 더 중요하다.

100ms 예측 오류는 괜찮지만, 500ms 예측 실행 시간은 그 자체로 새로운 latency다.
